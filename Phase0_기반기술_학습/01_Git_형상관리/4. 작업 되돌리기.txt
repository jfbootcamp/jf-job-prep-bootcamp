1. 작업 트리에서 수정한 파일 되돌리기 -- git restore
    1) 아직 스테이지에 올라가 있지 않은 수정사항을 되돌리기 
    2) 수정 사항을 취소하기 
    3) 스테이징하지 않은 상태에서 수정을 취소하고 원래대로 되돌리기 

2. 스테이징 되돌리기 :  git restore --staged   
    1) 수정된 파일을 스테이징까지 했을 때 스테이징을 취소하는 방법 
    2) 파일이 아직 스테이지에 올라가기 전(not staged)으로 돌아옴

3. 최신 커밋 되돌리기 : git reset HEAD^ 
    1) 수정된 파일을 스테이징하고 커밋까지 했을 때 
       가장 마지막에 한 커밋을 취소하는 방법 
       - 커밋 취소하면 커밋과 스테이징이 함께 취소됨 

4. git reset --soft HEAD^ 
    1) 커밋 취소하고 staged 상태로 작업 디렉토리에 보관함 

5. 특정 커밋으로 되돌리기 : git reset 해시  
    1) git reset --hard 복사한 커밋 해시     
    2) 특정 커밋으로 되돌릴 때 수정한 내용을 삭제해도 되는 경우 

6. 커밋 변경 이력 취소하기 -- git revert  
    1) 나중을 대비해서 커밋을 취소하더라도 커밋한 기록은 남겨 두어야 하는 경우 
    2) 변경 사항만 취소하고 커밋은 남겨 두는 경우 
    3) 예
        - 가장 최근에 커밋한 R5버전을 취소하고, R5 직전 커밋인 R2로 되돌아 가려고 함
        - 취소하는 R5 버전을 삭제하지 않고 남겨 두려고 함 
        - 이럴 때는 revert 명령을 사용하고 바로 뒤에 취소할 버전, 즉 R5의 커밋 해시를 지정함  

        - 로그에 R5를 revert한 새로운 커밋이 생김 
            - 기존의 R5 역시 사라지지 않았음
            - R5 버전을 지우는 대신 R5에서 변경한 내용만 취소하고, R5를 취소했다는 커밋을 새로 만든것임.
                - rev.txt 내용을 보면 R5 커밋에서 추가한 e가 취소된 것을 확인할수 있음.

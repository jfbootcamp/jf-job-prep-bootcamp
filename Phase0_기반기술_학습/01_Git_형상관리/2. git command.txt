1. 깃 저장소(repository) 만들기 
    1) git init 

2. 깃에서의 버전 관리 
    1) 버전
        - 문서를 수정하고 저장할 때마다 생기는 것
        - 버전을 만든 시간과 수정 내용까지 기록할 수 있음 
    2) DVCS            
        - 분산 버전 관리 시스템
        - 원래 파일 이름은 그대로 유지하면서 파일에서 무엇을 변경했는지를
          변경 시점마다 저장할 수 있음.
        - 버전마다 작업한 내용을 확인할 수 있고, 그 버전으로 되돌릴 수도 있음.

3. 깃의 작업 영역
    1) 작업 트리 (working tree)
        - 파일 수정, 저장 등의 작업을 하는 디렉토리임.
    2) 스테이지 (stage)
        - 스테이징 영역 (staging area)
        - 버전으로 만들 파일이 대기하는 곳임.  
    3) 저장소 (repository)               
        - 스테이지에서 대기하고 있던 파일들을 버전으로 만들어 저장하는 곳

4. 스테이지와 커밋
    1) hello.txt 파일을 버전으로 만들려면 우선 스테이지에 넣어야 함
        - 파일을 스테이지에 넣을 때 사용하는 명령은 git add 임.
    2) 스테이지에 있는 파일들은 버전을 만들 준비가 끝난 상태임.
    3) 깃에서 버전을 만드는 명령은 git commit임.  
        - 버전을 만든는 것을 '커밋한다'라고 함.
        - 깃에서 commit 명령을 실행하면 새로운 버전이 생성되면서 
          스테이지에 대기하던 파일이 모두 저장소에 저장됨.         

5. 작업 트리에서 변경 사항 만들기 
    1) 깃 상태를 확인하기 
        - git status               

    2) On branch main
        - 현재 main 브랜치에 있음.

    3) No commits yet
        - 커밋한 파일이 아직 없음

    4) nothing to commit
        - 현재 커밋할 파일이 없음 

6. 스테이지에 변경 사항 올리기 -- git add 
    1) 작업 트리에서 파일을 만들거나 수정했다면 스테이지에 추가함         
    2) 깃에서 버전 만들 준비를 하라고 알려 주는 것 (스테이징,staging)
    3) git add .
        - 파일 이름 대신 마침표(.)를 사용하면 작업 트리에 있는 수정 파일들을 
          스테이지에 한꺼번에 올릴 수 있음.
        - tracked 파일, untracked 파일 모두 스테이지에 올릴수 있음   

7. 스테이징한 파일 커밋하기 -- git commit 
    1) git commit 한 칸 띄운 후에 -m 옵션을 붙이고 커밋과 함께 저장할 메시지를 적음.
        - 커밋 메시지 

    2) git commit -m "message1"
        - 파일 1개가 변경 되었고, 파일에 1개의 내용이 추가되었다.
        - 스테이지에 있던 hello.txt 파일이 저장소에 추가된 것임.        

8. 버전이 제대로 만들어졌는지, 저장소에 저장된 버전 확인하는 명령어 -- git log          
    1) 커밋한 버전을 설명하는 정보가 나타남 
        - 커밋을 만든 사람과 시간, 커밋 메시지가 함께 보임.
    2) 커밋 기록을 자세히 살펴보기         

9. 스테이징과 커밋 한꺼번에 처리하기:  git commit -am
    1) git commit -am "message2"
        - 스테이징과 커밋 한꺼번에 처리함         

10. 변경 사항 확인하기 -- git diff         

11. git log 
    1) git log --stat
        - statics(통계)의 줄임말 
    2) git log --oneline
        - 한 줄에 한 커밋씩 보여줌 
    3) git log --oneline --branches 
        - 브랜치마다 최신 커밋 확인    
    4) git log --oneline --branches --graph
        - 브랜치와 커밋의 관계를 그래프 형태로 표시함

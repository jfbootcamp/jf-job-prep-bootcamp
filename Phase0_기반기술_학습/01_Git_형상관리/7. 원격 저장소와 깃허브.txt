1. intro
    1) 지금까지 자신의 컴퓨터에서 작업한 뒤 같은 컴퓨터 안에 커밋을 함 
        - 이 저장소를 지역 저장소(local repository)라고 함.

2. 원격 저장소 
    1) remote repository를 연결해서 버전 관리하는 파일을 쉽게 백업함 
    2) 지역 저장소가 아닌 컴퓨터나 서버에 만든 저장소를 말함.
    3) 백업, 협업

3. 깃허브 
    1) 원격 저장소에서 깃을 사용함 
    2) 지역 저장소를 백업할수 있음
    3) 온라인 개발 툴 사용할 수 있음
        - AI 동료
    4) 협업 프로젝트에 사용할 수 있음
    5) 자신의 개발 이력을 남김할 수 있음 
    6) 다른 사람의 코드를 살펴볼 수 있고, 오픈 소스에 참여할 수도 있음  

4. 지역 저장소를 원격 저장소에 연결하기   
    1) 지역 저장소 만들기
    2) 원격 저장소 연결하기 
        - git remote add origin 주소 
                     remove          
        - git remote -v 

5. 지역 저장소와 원격 저장소 동기화하기 
    1) 원격 저장소에 커밋 처음으로 올리기 
        - git push -u origin main        
            - origin(원격 저장소의 별칭)의 main 브랜치로 푸시하라는 명령 
            - -u 옵션
                - 지역 저장소의 브랜치를 원격 저장소의 브랜치에 연결하는 것
                - 처음에 한 번만 사용하면 됨 

    2) 원격 저장소에 파일 옭리기 
        - git push                         

    3) 원격 저장소에서 직접 커밋하기 

    4) 원격 저장소에서 커밋 내려받기 - git pull 
        - 원격 저장소와 지역 저장소의 상태가 같아지도록 원격 저장소의 커밋을 
          지역 저장소로 가져옴 
        - git pull origin main

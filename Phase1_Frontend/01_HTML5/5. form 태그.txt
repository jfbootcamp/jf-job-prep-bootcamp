1. form 태그 주요 속성
    1) action
        - 폼 데이터 전송할 서버 URL
            - 현재 상태: 백엔드 서버가 없어서 404 에러 발생
            - 실제 서버 엔드포인트 URL로 연결
    2) method
        - 데이터 전송 방식 
        - POST: 데이터가 URL에 노출되지 않음 (보안에 유리)
        - GET: 데이터가 URL에 표시됨 (?key=value 형태)            

    3) name
        - 폼의 이름 (JS에서 접근할때 사용)     
        - 현재 상태 
            - [문의하기 버튼 클릭]
                => 브라우저가 /submit으로 POST 요청
                => 서버 없음 => 404 Not Found 에러 
        - 벡엔드 연동 시 상태
            - HTML Form  => /submit => 벡엔드 서버  =======> DB 저장
              (프런트)        (URL)     (Java, Node.JS      (MySQL,Oracle등)
                                        , Python)

2. 입력 필드 
    1) label의 for 속성은 input의 id와 연결됨 
        - 레이블 클릭 시 해당 입력 필드로 포커스 이동 (접근성 향상)

    2) input 속성들
        - type="text"   : 한 줄 텍스트 입력 필드 
          type="email"  : 이메일 형식을 자동으로 검증함 (@ 포함 여부)
                          모바일에서 @ 키가 포함된 키보드가 표시됨 
                          유효하지 않은 형식 입력 시 제출 차단    
          type="tel"    : 전화번호 입력 전용 필드
                          모바일 기기에서 숫자 키패드가 자동으로 표시됨 
                          형식 검증은 자동으로 되지 않음      

        - id            : 요소의 고유 식별자 (label의 for와 연결)
        - name          : 서버로 전송될 때 사용되는 데이터 키(key)
        - required      : 필수 입력 항목 (비어있으면 제출 불가)
        - placeholder   : 입력 전 보여지는 힌트 텍스트 (입력시 사라짐)

3. 드롭다운 메뉴 
    1) select 태그 
        - 드롭다운 목록
        - 여러 옵션 중 하나를 선택하는 UI 
            - option 태그로 선택 항목을 정의 
                - value 속성 : 서버로 전송되는 실제 데이터 

4. 여러 줄 텍스트 입력 
    1) textarea       
        - input과 달리 여러 줄의 텍스트를 입력받을 수 있음 
        - rows : 기본으로 보이는 줄 수 (높이)
        - cols : 기본으로 보이는 너비 
        - 기본값은 태그 사이에 입력 (value 속성 아님)         

5. 제출 버튼 (button) 
    1) 클릭 시 form의 action의 URL로 데이터가 전송됨 
    2) type 속성 
        - submit : 폼 제출 (기본값)
        - reset : 폼 초기화 (모든 입력값 삭제)
        - button : 기본 동작 없음 (JS로 기능 구현)

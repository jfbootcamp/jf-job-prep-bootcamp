1. 개요 
    1) CSS 레이아웃 기술은 웹의 발전과 함께 3세대에 걸쳐 진화했음
        - 1세대 : float, position, table (1990년대 ~ )
        - 2세대 : Flexbox                (2012 ~ )
        - 3세대 : Grid                   (2017 ~ )

2. 1세대 : float, position, table
    1) 원래 레이아웃 용도로 만들어진 것이 아님
    2) float: 이미지 주변에 텍스트를 감싸기 위한 용도 
    3) table: 표 데이터를 표시하기 위한 용도 
    4) position: 요소의 위치를 미세 조정하기 위한 용도 
    5) 문제점
        - 수직 중앙 정렬 매우 어려움 
        - 동일 높이 컬럼 구현 복잡 
        - 코드가 복잡하고 유지보수 어려움        

3. 2세대 : Flexbox (2012 ~ )
    1) 1차원 레이아웃 (가로 OR 세로, 한 방향)           
    2) 정렬(alignment)에 특화
    3) 유연한 크기 조절 
    4) 장점
        - 수직 중앙 정렬이 쉬워짐
        - 동일 높이 컬럼 자동 처리
        - 아이템 순서(order) 변경 가능 
        - 반응형 레이아웃 구현 용이 
    5) 한계
        - 2차원 레이아웃(행,열 동시 제어) 구현이 어려움         
        - 복잡한 그리드 레이아웃에 부적합

4. 3세대 : Grid (2017 ~ )
    1) 특징 
        - 2차원 레이아웃 (가로 and 세로 동시 제어)
        - 복잡한 레이아웃을 간단하게 구현 
        - 레이아웃 전용으로 설계됨 

5. Flexbox vs Grid
    1) Flexbox가 적합한 경우 
        - 네비게이션 바
        - 카드 내부 레이아웃
        - 버튼 그룹 
        - 아이템 정렬 (수평/수직)
        - 단순한 1차원 배치 
    2) Grid가 적합한 경우
        - 페이지 전체 레이아웃 
        - 대시보드/관리자 페이지 
        - 이미지 갤러리 
        - 카드 그리드 (상품 목록)
        - 복잡한 2차원 레이아웃 
    3) 이 둘은 대체 관계가 아니라 보완 관계임 

4. 결론
    1) Grid : 페이지의 뼈대 (전체구조)
    2) Flexbox : 컴포넌트 내부 정렬 (세부조정)
    3) 현대 웹 개발에서는 Grid와 Flexbox를 함께 사용하는 것이 표준임

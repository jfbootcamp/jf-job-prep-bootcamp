1. 소스코드 평가와 실행
    1) JavaScript 엔진은 코드를 실행하기 전에 소스코드 평가 단계를 거친 후
       실행 단계에서 코드를 수행함 
    2) 평가 단계에서 변수, 함수의 선언문만 먼저 실행하여
       실행 컨텍스트가 생성되고, 변수와 함수 선언이 메모리에 등록됨

2. 실행 켄텍스트 
    1) 변수, 함수, 스코프, this 등의 정보를 담고 있음 
    2) 코드가 실행되는 동안 콜 스택에 쌓이며, 새로운 함수를 만나면
       기존 실행을 잠시 중단 후 그 함수 내부로 진입
    3) 이후 진입한 함수의 평가 및 실행 과정을 거치게 됨
    4) 그 과정에서 새로운 해당 켄텍스트가 생성되고,
       함수 실행이 끝나면 해당 컨텍스트가 제거됨 
    5) 전역 실행 컨텍스트, 함수 실행 컨텍스트가 존재   

3. 콜 스택
    1) JavaScript 엔진이 함수 실행 순서를 관리하는 스택 자료구조
    2) 실행 컨텍스트의 콜 스택에 의해 코드의 실행 순서가 결정됨
        - 함수가 호출되면 함수 실행 컨텍스트가 순차적으로 콜 스택에 푸시되어 순차적으로 실행됨
        - 가장 최근에 추가된 함수가 가장 먼저 실행 완료(LIFO)되어 스택에서 제거됨 
    3) 가장 마지막에 추가된 실행 켄텍스트가 먼저 제거됨 
    4) 함수가 호출되면 실행 켄텍스트가 생성되어 스택에 쌓이고,
       함수 실행이 끝나면 제거됨   

4. 실행 컨텍스트(context) 종류 
    1) 전역 실행 컨텍스트 
        - 코드 실행이 시작되면 자동으로 생성
        - 전역 객체(window 또는 globalThis)와 전역 스코프를 포함 
        - 전역 실행 컨텍스트는 프로그램 종료 시점까지 유지 
    2) 함수 실행 컨텍스트 
        - 함수가 호출 될때마다 새로운 실행 컨텍스트가 생성 
        - 함수 내부에서 사용되는 변수와 스코프를 분리 

5. 자바스크립트 런타임 
    1) 런타임
        - 프로그램이 실제 실행되는 시간과 공간 
        - 코드가 단순히 존재하는 것이 아니라, 실행되기 위해 필요한 실행 맥락

    2) 런타임 환경
        - 특정 언어가 실행되기 위해 제공되는 실행 도구 모음 
        - 자바스크립트의 경우: 엔진 + API + 이벤트 루프 등으로 구성됨 
            - 엔진 : 자바스크립트 코드를 해석하고 실행하는 핵심
            - API : 런타임 환경에서 제공하는 기능 집합 (Web API, Node.js API)  
            - 이벤트 루프 : 동기, 비동기 작업의 실행 순서를 조율하는 메커니즘  
        - 언어만으로는 불완전 => 반드시 런타임 환경이 필요함      

6. 싱글 스레드 
    1) 한 번에 하나의 작업만 처리할 수 있는 구조, 단위
    2) JavaScript는 기본적으로 싱글 스레드 언어로, 
       하나의 콜 스택을 사용하여 작업을 순서대로 처리함 
    3) 하나의 작업이 완료될 때까지 다음 작업이 실행되지 않고 대기함                                                                   
    4) 특징 
        - 코드를 직관적이고 예측 가능하게 실행함 
        - 하나의 긴 직업이 끝나지 않으면 나머지 작업들이 모두 차단되는 단점이 있음

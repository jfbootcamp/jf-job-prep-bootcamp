1. 동기처리
    1) 코드를 순차적으로 실행하는 방식 
    2) 앞의 작업이 완료될 때까지 다음 작업은 실행되지 않고 대기함 
    3) 직관적이고 코드 흐름을 이해하기 쉬우나, 오래 걸리는 작업이
       전체 프로그램을 차단할 수 있음 

2. 비동기처리
    1) 특정 코드의 실행이 끝날 때까지 기다리지 않고 
       다음 코드를 즉시 실행하는 방식 
    2) 오래 걸리는 작업(네트워크 요청, 타이머 등)을 백그라운로 넘기고,
       그동안 다른 작업이 실행됨
    3) 코드 흐름이 멈추지 않아 전체 프로그램의 성능이 향상됨 

3. setTimeout() 함수
    1) 자바스크립트에서 제공하는 타이머 함수
    2) 특정 시간이 지난 뒤에, 전달한 콜백 함수를 실행      

4. 콜백(Callback) 정리
    1) 비동기 작업이 완료되면 실행할 함수를 미리 전달 
    2) 서버 API 호출 시뮬레이션
        - setTimeout(()=> callback(data), 지연시간)
    3) 순서 보장
        - 콜백 안에서 다음 작업 호출 

5. 실무 포인트 
    1) 이벤트 리스너, Node.js API에서 여전히 사용 
    2) 일반 API 호출은 Promise/async-await 권장

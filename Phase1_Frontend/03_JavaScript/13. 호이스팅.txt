1. 호이스팅 (Hoisting)
    1) 변수와 함수 선언이 코드 실행 전에 스코드 최상단으로 "끌어올려지는" 현상
    2) JavaScript 엔진이 코드 실행 전에 선언문을 먼저 처리하여,
       마치 선언이 스코프의 최상단으로 끌어 올려진 것처럼 동작하는 현상 
    3) 호이스팅은 선언이 물리적으로 이동하는 게 아니라,
       JS엔진의 2단계 실행 (생성 단계, 실행 단계)처리로 인해 발생하는 현상임.   

2. 코드로 설명 
    1) 우리가 작성한 코드 

    // 1. 함수를 먼저 호출
    sayHello();     

    // 2. 그 다음 함수 선언
    function sayHello() {
        console.log("안녕!");
    }       

    2) JS 엔진이 해석하는 코드 

    // 1. 함수 선언이 먼저 처리됨 
    function sayHello() {
        console.log("안녕!");
    } 

    // 2. 그 다음 코드 실행    
    sayHello();         // 정상 호출

    - 호이스팅으로 함수가 먼저 등록됨

3. 변수 호이스팅 (var 키워드)
    1) var 변수는 선언만 호이스팅되고, undefined로 초기화됨 
    2) var 변수는 선언(declaration)만 호이스팅,
       할당(assignment)은 제자리!

4. 코드로 설명        
    1) 우리가 작성한 코드 

        console.log(num);       // 선언 전에 접근!

        var num = 123;

        console.log(num); 

    2) 호이스팅 후 실제동작 
        var num;                // 선언만 호이스팅 
        console.log(num);       // undefined
        num = 123;              // 할당은 제자리 
        console.log(num);       // 123

    3) var 의 문제점 
        - 선언 전 접근해도 에러 없음 ==> 버그 발견 어려움!
